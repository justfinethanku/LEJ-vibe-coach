# Open-Source Roadmap

## Phase 1: Reusable local engine

Already underway.

Includes:

- research import into SQLite
- lesson and quiz storage
- browser UI
- progress tracking
- learner comments
- a project context file for repo adaptation

## Phase 2: AI-assisted repo adaptation

Next major milestone.

Goal:

An AI should be able to enter a repo and produce the first useful Vibe Coach setup quickly.

Includes:

- repo-level AI instructions and adaptation contract
- structured project blueprint for inspection and curriculum framing
- repo inspection workflow
- research doc generation
- project context updates
- first-pass lesson and quiz generation

## Phase 3: Personalization engine

Goal:

Make the curriculum respond to the learner instead of staying static.

Includes:

- concept-gap detection from quiz misses
- lesson revision suggestions from comments
- requests for examples or deeper explanations
- learner profile and preferred teaching style

## Phase 4: Project onboarding product

Goal:

Make Vibe Coach genuinely strong for teams and open-source maintainers.

Includes:

- fresh onboarding packs for new contributors
- domain-specific learning tracks
- concept maps across a codebase
- handoff and orientation flows for unfamiliar repos

## Product Rule

Do not chase “AI magic” first.

The valuable path is:

1. reliable repo context
2. strong research
3. explicit curriculum
4. measured personalization

That sequence keeps the project useful and maintainable.
