# Product Vision

# Vibe Coach

The educational platform that teaches you while you build.

## Why This Should Exist

Most onboarding and learning systems fail for one of two reasons:

- they are generic and detached from the real codebase
- they are project-specific but go stale fast

Vibe Coach should solve both problems.

It should:

- learn the repo
- turn that context into a lesson path
- help the learner build while learning
- adapt over time based on what the learner does and does not understand

## Best Use Cases

- onboarding new people to an existing project
- learning a domain that is outside your background
- helping solo builders ramp into complex systems faster
- preserving architectural intent as a project changes

## Product Promise

You should be able to drop Vibe Coach into a repo and get:

- project research
- a guided learning path
- quizzes that test understanding
- learner notes that improve future lessons

## Open-Source Angle

The useful open-source artifact is not just the UI.

It is the schema:

- the repo contract
- the database contract
- the AI adaptation contract
- the curriculum generation workflow

If those are good, the system becomes genuinely reusable.

## What “Amazing” Means Here

This should not feel like an LMS with repo-flavored wallpaper.

It should feel like:

- onboarding that understands the codebase
- documentation that teaches
- AI assistance that compounds over time

That is a much better product.
